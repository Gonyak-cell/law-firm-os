# G5-C Invoice Tax Billing UI Report

TUWs: `LFOS-G5-W07-T011` through `LFOS-G5-W07-T016`

Branch: `codex/lawos-g5-invoice-tax-billing-ui`

Base: `codex/lawos-g5-wip-prebill-adjustment`

This slice does not claim G5 runtime readiness. G5-C adds synthetic-only
descriptor evidence proving that invoice issue requires idempotent issue
evidence, invoice-line generation requires WIP-to-invoice reconciliation,
TaxInvoice evidence covers issue/transmit/fail outcomes, issued invoices move
through correction workflows with direct edit blocked, Billing UI states enforce
role-based detail masking, and G5 Billing closeout records time-to-invoice
evidence without opening runtime writes, database rows, audit appends, API
handlers, invoice issue services, invoice-line generation services, GL posting,
issued-invoice mutation, real client data, or billing runtime services.

## Scope

G5-C depends on G5-B WIP PreBill Adjustment evidence and the G5 Billing/Finance
entry plan. It covers invoice issue, invoice-line reconciliation, TaxInvoice
state, issued-invoice correction, Billing UI masking, and Billing closeout
evidence while preserving the RP12 descriptor-only no-runtime contract.

| File | Purpose |
| --- | --- |
