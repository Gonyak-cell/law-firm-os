## Phase 개요

- **목표:** Amplitude 패리티 데모 셸을 실제 법률 제품 화면군으로 전환한다. '연결 작업'이 아니라 '제품 빌드' 규모(50개 flow group 중 1개만 라이브, 자동 테스트 0 — apps/web/package.json에 test 스크립트 부재 실측).
- **진입조건:** L2의 인증(WP:LT-L2-W02)·서버측 권한·Wave 1 write API(WP:LT-L2-W03) 일부 가동 — 화면 단위 incremental 착수. staging 가동(WP:LT-L3-W02)은 **완료 게이트(E2E)의 전제**이며 staging 의존은 해당 TUW(LT-L4-W01-T21)에만 바인딩한다.
- **완료 게이트(L4-EXIT):** Wave 1 화면 전부 staging에서 합성 시드로 E2E(로그인→matter 생성→문서 filing→work queue→audit 조회) [→ LT-L4-W01-T21] + UI 회귀 green [→ LT-L4-W04-T04] + npm test+build green [→ LT-L4-W01-T21, LT-L4-W04-T04].
- **granularity: 전 WP provisional** — `rebaseline_after: WP:LT-L0-W02`. LT-L0-W02(런타임 격차 보고서) 완료 후 본 phase의 TUW는 재분해 의무가 있으며, 기존 ID는 withdrawn 처리 후 신규 발번한다(재번호 금지).
- 원문 매핑: 작업 묶음 1→W01, 2→W02, 3→W03, 4→W04, 5→W05, 6→W06. 산출물 매핑: "Wave 1 제품 UI"→W01, "UI 회귀 스위트"→W04, "교육 원천 문서"→W06.
