공통 사항(전 TUW, §0 외 추가 주의 — 복제 아님):

- DB 제품·호스팅 스택은 LT-L1-W06 결정에 종속. 아래 `packages/persistence/` 경로는 가칭 — 신규 패키지 디렉토리이며, 기존 `packages/` 파일 수정이 필요한 TUW만 `제약 예외`에 명시한다.

#### LT-L2-W01-T02 — 재기동 내구성과 동시 append 직렬화를 보장한다
- 유형/실행/리스크/가중치: runtime_write / agent_implementation / A / M
- 목표: 프로세스 재기동 후에도 tenant별 해시체인이 끊기지 않고 이어지며, 동시 append 경합에서 sequence 갭·중복이 발생하지 않는다.
- 산출물: `packages/persistence/src/audit-event-store.js` 직렬화 로직 보강, `packages/persistence/test/audit-event-store-durability.test.js`(신규)

- 선행: LT-L2-W01-T01
- 완료 기준:
  1. append N건 → 프로세스 종료 → 재기동 → 추가 append 후 verifyHashChain `{ok:true}`(재기동 왕복 테스트 green).

- 게이트: L2-EXIT
- 권한·감사 영향: 감사 체인 연속성 보장 강화 — 권한 모델 변화 없음. 비약화 논증 필요.

#### LT-L2-W01-T07 — 쓰기 트랜잭션 경계와 원자적 audit 결합을 구현한다

- 참조: workbook/matter-post-cp-launch-plan.md §5 L2-1·L2-3; workbook/matter_dev_docs/22_API_이벤트_통합_명세.md §6-6(partial state 방지); contracts/runtime-readiness-contract.json(RTG-003)
- 선행: LT-L2-W01-T01, LT-L2-W01-T06
