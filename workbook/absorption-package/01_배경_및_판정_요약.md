# 01. 배경 및 판정 요약

## 1. 왜 이 패키지가 필요한가

2026-06-11, 별도 세션(Codex)에서 작성된 matter 개발문서 패키지(Plan A, 25개 문서)가 도착했다. 기존 리포 체계(Plan B)와 8개 차원 × 적대적 검증으로 비교한 결과:

| 차원 | A | B | 승자 |
|---|---|---|---|
| 범위 완전성 | 6 | 7.5 | B |
| 분해 품질·실행 가능성 | 4 | 7 | B |
| 검증·게이트 체계 | 3.5 | 7.0 | B |
| 리스크·보안·컴플라이언스 | 6 | 8 | B |
| **아키텍처·기술 구체성** | **6.5** | **5.5** | **A** |
| 개발 거버넌스·운영 모델 | 4.5 | 7.5 | B |
| 적응성·피드백·진화 | 4 | 7.0 | B |
| 전략·목표 정합성 | 5 | 7 | B(조건부) |

**판정: B를 마스터 실행 체계로 유지한다.** A로 갈아타면 착수 가능 단위 0개 체계(TUW 204건 전건 보일러플레이트, ID 자기모순)로 이행하면서 닫힌 팩 320+개·리뷰 영수증 144개·validator 28개를 고아화하고, 소유자 표준 목표문("closeout-pack plan을 권위적 실행 소스로")을 위반하게 된다.

**그러나 A에는 B가 갖지 못한 실질이 있다.** 본 패키지가 흡수하는 것:

1. **M365 기술 결정** (A의 유일 승리 차원): Office.js+MSAL/NAA, Smart Alerts send mode 3종, Graph 3대 ID, 조직 배포 — Microsoft Learn 공식 문서와 교차검증 전 항목 일치. B의 RP08/RP22에는 플랫폼 기술 결정이 0건.
2. **런타임 게이트 의미론** (A의 CG-002~006): B의 최대 실질 약점 — 닫힌 팩의 약 절반(154/323), 최근 119개 연속 팩이 descriptor-only인 채 production_ready로 닫혀 라벨 의미가 희석됨 — 을 정조준.
3. **HR 스펙** (A 문서 11): 16객체, rule-engine-first, 민감정보 가드. B의 HRX 901은 실물 0인 차용증 상태.
4. **통합 리스크 레지스터 + 전략 선언문**: A의 RISK-001~012/OQ-001~015와 단일 전략 선언 구조 — B에는 분산돼 있거나 부재.

## 3. HR 감사 결론 (소유자 질의 회신)

**HR은 Plan B에 "기계 강제되는 약속"으로는 확실히 존재하고, 실물은 0이다.**

존재하는 레이어 (전부 직접 확인):
- 목표문·총량 회계: 54,355 + HRX 901 = 55,256. CP00-001(2026-06-06)부터 **모든 팩 manifest에 901 일치를 `validate-closeout-pack.mjs:118-125`가 강제** — 322+개 전 팩 보유.
- 경계 선언: `splits_hrx_product:false` 85건 + `hrx_embedded_boundary` 51건 (코드·계약·문서 230여 곳).
- 완료 차단 게이트: 실행계획 §8이 "901 HRX units tracked"를 최종 완료 조건으로 요구 — **HRX 없이 제품 완성 선언 불가능.**

부재하는 레이어:
- spec requirement ledger 227건에 HR 0건 — **원천 스펙(6/3 첨부 텍스트 144행)이 "완전한 HR/급여시스템"을 명시 제외**했고, 사흘 뒤(6/6) 목표문이 901을 총량에만 주입.
- RP 프로그램 0, weighted ledger unit 0, CP 팩 0, HR 계약 0, 코드 0. v2.0 사양서 docx(763행)에도 HR 0건. HR 본문 스펙은 Plan A 문서 11에만 존재.

핵심 기술 발견 2건 (문서 03의 설계 근거):
1. **901은 기존 가중 공식으로 도달 불가** — 전수 시뮬레이션 결과 최근접 895. 등재는 "공식(complexity 2.2 + high-risk = 895) + 명시 보정 테이블(+6, HR 민감가드 보강 명목)"으로만 가능하며 보정을 ledger에 정직하게 박제해야 한다.
2. **901은 동결 상수** — 변경하면 닫힌 322+팩이 소급 invalid. HRX 전개는 정확히 901 unit이어야 한다.
