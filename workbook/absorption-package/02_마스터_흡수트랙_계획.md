# 02. 마스터 흡수트랙 계획 — `docs/matter-pack-integration/`

Plan A 전체(25개 문서)를 Plan B 요구사항 소스로 흡수하는 트랙. LDIP/V2 표준 절차의 하이브리드 변형. 모든 산출물은 planning-only(validator 비독해 경로).

## 0. 설계 전제

- **베이스라인 스냅샷**: CP00-323 닫힘 / 다음 unit RP10.P02.M09.S03 (2026-06-11 기준 — **실행 시 재확인 필수**). 단, 전략 선언문에는 커서 숫자를 넣지 않는다(execution plan §2의 CP00-146 스테일 부패 반복 금지).
- **ID 네임스페이스** (충돌 grep 0건 확인 완료): 요구 `MAT-REQ-<FAMILY>-###`, 리뷰 `C-MPACK-01~05`, 리스크 `UPR-###`, 오픈 질문 `UPQ-###`, 인간 결정 `MAT-DEC-0N`. Plan A 원문의 `MAT-OUT-*-TUW-*` 패턴은 **재사용 금지**(TUW 어휘 오염 방지).
- **거버넌스 비흡수 결정** (3중 기록: 통합계획 §9 + adjudication §3 + overlay JSON `governance_absorption_policy`): Plan A의 TUW 204/L0~L13/R0~R14/P0~P14/VC·CG ID/Ledger 5종/4역할 agent는 체계로 흡수하지 않는다. 요구사항만 추출한다. 근거: plan-vs-plan-verdict 7승 1패. 거버넌스 문서(02·03·15~20·24)는 governance family로 분리해 **등가성 입증** 또는 reject_with_reason 처리.

## 2. 산출물 B — 통합 리스크 레지스터 (json+md 쌍)

- entries: `UPR-###` — Plan A RISK-001~012 전건 시드(원 ID는 source_refs 보존). 예: RISK-002(User/Employee 동일시) → UPR-002 {P0_BLOCKER, needs_human_decision, MAT-DEC-02, first_anchor: RP11}.
