# Matter Core

Matter Core owns the RP05 Matter domain foundation for Law Firm OS.

CP00-177 establishes the first `Matter`, `MatterWiki`, and provider-neutral `MatterGraph` skeleton surfaces. It is intentionally non-runtime: the package defines model descriptors, synthetic fixtures, validation packets, and no-write evidence only. It does not implement a graph provider, Citation Ledger runtime, AI workflow, client-visible wiki publishing, or Loop engine behavior.

CP00-178 advances that foundation into a descriptor-only Matter service/domain boundary. The service entrypoint normalizes requests and returns deterministic no-write descriptors for Matter opening, member assignment, task planning, wiki section staging, and graph relationship staging. It does not persist records, evaluate runtime permissions, write audit events, acquire locks, dispatch approvals, issue network calls, expose client-visible wiki output, or implement Loop execution.

CP00-179 keeps the CP00-178 service boundary as the source service and adds the service-tail evidence/review bridge. It validates golden descriptor outcomes, negative safe-error cases, retry/rollback determinism, and the hardened pack-level Claude review sequence. Claude remains a read-only reviewer only; it is not final approval and cannot create an enterprise trust claim.

CP00-180 is a small Risk A continuation for sensitive service prechecks. It keeps tenant boundary, matter trace, permission, audit hint, idempotency, lock, and persistence evidence descriptor-only and fail-closed while preserving the CP00-178 source service and CP00-179 evidence bridge.

The v2 overlay is treated here as the knowledge/evidence spine:

- `MatterWiki` is a first-class Matter knowledge workspace, not a free-form note.
- `MatterGraph` is provider-neutral and permission-trimmed by contract.
- Citation Ledger dependencies are reserved for RP06 and later.
- Future Loop context references are stable but inert until the RP18 Loop stage.
