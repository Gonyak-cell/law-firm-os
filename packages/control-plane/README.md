# Control Plane

`packages/control-plane` owns the RP00 Product Constitution And AI Control Plane surface for Law Firm OS.

This package starts as a contract-first module. Its first executable subphase, `RP00.P00.M00.S01`, inventories the acceptance sources, authority boundaries, and blocked claims that must hold before implementation work can be called production-ready. `RP00.P00.M01.S01` drafts the contract shell for `ProductContract`, `AIControlRule`, `HermesGate`, `ClaudeReviewGate`, `HumanApproval`, and `BlockedClaim`; `RP00.P00.M02.S01` defines their type/shape registry; `RP00.P00.M03.S01` locks lifecycle gates for plan authoring, AI implementation handoff, Hermes validation, Claude review, human approval, blocked claims, and subphase closeout; `RP00.P00.M04.S01` defines Matter-first trace rules for tenant-scoped control-plane records when client, document, finance, external-share, portal, or client-facing AI-output data is touched; `RP00.P00.M05.S01` defines the minimum permission decision and audit evidence baseline required before control-plane behavior can run; `RP00.P00.M06.S01` requires every control-plane example or fixture to be synthetic-only, explicitly marked, and free of real client, matter, document, finance, credential, or secret data; `RP00.P00.M07.S01` inventories the no-write validation command matrix surfaces used for closeout evidence; `RP00.P00.M07.S02` defines acceptance gates for command evidence, H00 command id parity, Claude review metadata, and construction inspection; `RP00.P00.M07.S03` locks non-goal boundaries so validation evidence cannot approve generation output, real-data import, production mutation, or all of M07/RP00; `RP00.P00.M07.S04` maps the target contracts, validators, tests, Hermes wiring, package manifest, and closeout evidence roots for the validation command matrix; `RP00.P00.M08.S01` inventories the H00 preflight fields Hermes records before implementation advances beyond the contract and acceptance baseline; `RP00.P00.M08.S02` defines acceptance gates for H00 preflight field presence, command receipts, blocked claims, gate outcome, no-real-data/no-write attestations, Claude dependency, construction inspection, and next-boundary evidence; `RP00.P00.M09.S01` inventories the C00 design brief scope for architecture, security, missing-test, validation-theater, AI-write, and go/no-go review questions; `RP00.P00.M09.S02` defines acceptance gates for C00 design brief packet parity, severity policy, go/no-go policy, evidence fields, Claude review metadata, and construction inspection; `RP00.P00.M10.S01` inventories the RP00.P00 handoff scope, closes weighted-title generator-alignment deferrals as contract-title overrides, and records the next boundary into `RP00.P01.M00.S01`; `RP00.P01.M00.S01` creates the control-plane package directory layout with `src`, `test`, and `fixtures` surfaces before entity models are implemented in later Domain Model subphases; `RP00.P01.M01.S01` implements the ProductContract model with stable identifiers, tenant scope, status lifecycle, nullable Matter references, and audit metadata; `RP00.P01.M02.S01` implements the AIControlRule model with action enums, forbidden hard-stops, review-required triggers, blocked-claim references, and audit metadata; `RP00.P01.M02.S02` implements the AIControlRule primary entity identifier policy for canonical `air_` lowercase snake-case rule IDs; `RP00.P01.M02.S03` implements the AIControlRule Tenant scope field for canonical `lfos_` tenant IDs and same-tenant context checks; `RP00.P01.M03.S01` opens the HermesGate Package directory layout without claiming HermesGate model completion; `RP00.P01.M03.S02` implements the HermesGate Primary entity identifier policy for canonical `hg_` lowercase snake-case gate IDs; `RP00.P01.M03.S03` implements the HermesGate Tenant scope field for canonical `lfos_` tenant IDs and same-tenant context checks; `RP00.P01.M03.S04` implements the HermesGate Matter trace reference for nullable firm-level gates and canonical `matter_` Matter references when a Matter context is present; `RP00.P01.M03.S05` implements the HermesGate Lifecycle status enum using shared control-plane lifecycle states while deferring the transition map to `RP00.P01.M03.S10`; `RP00.P01.M03.S06`
